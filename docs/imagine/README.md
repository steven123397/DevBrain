# DevBrain 构想存储库

文档状态：imagination index
最后更新：2026-04-23
适用范围：高于当前版本路线的远期设想、概念笔记、讨论摘录与尚未收敛的产品想象

## 1. 定位

`docs/imagine/` 用来保存尚未进入正式设计、计划或状态流的远期构想。

这里的内容可以大胆、开放、未定型，但默认不构成当前产品边界、实现计划或验收标准。

## 2. 与正式文档的关系

- 成熟前：只作为想象、讨论与概念沉淀。
- 成熟后：再提炼到 `docs/design/`，形成长期边界、关键取舍与正式设计。
- 启动实现前：再进入 `docs/plan/`，拆分任务、完成定义与验证方式。
- 推进过程中：实时进度仍只写入 `docs/status/`。

## 3. 记录原则

- 保留想法的上下文，而不是过早包装成结论。
- 区分「已形成共识」和「待继续讨论」。
- 可以记录外部参考，但不把对标对象当成产品目标。
- 避免把远期想象直接反向污染当前 MVP 主闭环。
